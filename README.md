## Контрольные вопросы

1. Чем Minimal API отличается от MVC в ASP.NET Core?

Minimal APIs фокусируются на упрощении разработки API за счёт минимального количества кода и явного определения маршрутов. Они не используют контроллеры, а определяют эндпоинты напрямую в файле Program.cs. MVC  разделяет приложение на три компонента: модель, представление, контроллер 

2. Что произойдёт, если поставить app.MapGet(...) до app.Use(...)?

Middleware не будет выполняться для этого маршрута.

3. Почему ASP.NET Core преобразует PascalCase в camelCase при сериализации 
JSON?

ASP.NET Core преобразует PascalCase в camelCase при сериализации JSON по умолчанию из-за настроек сериализатора по умолчанию. Это решение было принято с целью соответствия общепринятым стандартам в индустрии, в частности, конвенциям JavaScript и TypeScript, где camelCase является распространённым форматом для имён свойств объектов.

4. Что означает код ответа HTTP 401? А 404? А 200?

401 Unauthorized — код означает, что для доступа к ресурсу требуется аутентификация, но она не была предоставлена или завершилась неудачно.
404 Not Found — сервер не может найти запрошенный ресурс.
200 OK — стандартный успешный ответ для большинства запросов.

5. Чем dotnet run отличается от dotnet watch run?

dotnet run — это команда для запуска приложения из исходного кода одной командой. Она автоматически компилирует код, генерирует выходные файлы и запускает программу. 

dotnet watch run - при использовании этой команды сервер будет автоматически перезапускаться при изменении файлов
